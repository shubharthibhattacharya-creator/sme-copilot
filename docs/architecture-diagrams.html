Auth Flow
1. User signs in via Clerk
2. Clerk issues JWT (session token)
3. Next.js middleware validates JWT
4. companyId extracted from Clerk metadata
5. Injected into every API request as header
6. NestJS @CurrentUser() decorator provides it
7. Every DB query scoped to companyId
